# Configuration

## config.php

The main configuration file is `core/config.php`, generated during installation. It contains:

```php
<?php
define('DB_HOST', 'localhost');
define('DB_NAME', 'your_database');
define('DB_USER', 'your_username');
define('DB_PASS', 'your_password');
define('DB_PREFIX', 'c3_');
define('SITE_URL', 'https://yoursite.com');
define('AUTH_SALT', 'random_64_char_hex_string');
```

> **Never commit this file to version control.** It contains database credentials.

## .htaccess

The `.htaccess` file is generated by the installer with the correct `RewriteBase` for your installation path. It handles:

- URL routing to the front controller
- Admin panel routing
- Blocking direct access to `core/` files
- Preventing PHP execution in the uploads directory
- Passing through real files (CSS, JS, images) directly

The format follows the same pattern as WordPress:

```apache
<IfModule mod_rewrite.c>
RewriteEngine On
RewriteBase /

RewriteCond %{REQUEST_FILENAME} !-f
RewriteRule ^admin(/.*)?$ admin/index.php [L,QSA]

RewriteCond %{REQUEST_FILENAME} !-f
RewriteRule . index.php [L]
</IfModule>
```

## Admin Settings

Most site configuration is managed through the admin panel under **Settings**:

| Tab | Settings |
|---|---|
| **General** | Site name, tagline, description, timezone, date format |
| **Reading** | Homepage display (latest posts or static page), posts page, posts per page |
| **Comments** | Enable/disable comments, require moderation |
| **Users** | Public registration, default role |
| **Email** | Mail method (PHP mail or SMTP), SMTP credentials |
| **SEO** | Meta description, meta keywords |

## Static Front Page

Under **Settings → Reading**, you can set a static page as your homepage instead of the blog listing:

1. Create a page in **Admin → Pages**
2. Go to **Settings → Reading**
3. Change "Your homepage displays" to **A static page**
4. Select your page from the **Homepage** dropdown
5. Optionally set a **Posts page** — this page's URL will show the blog listing

The front-page template (`front-page.php`) is used for the homepage. If the page content starts with `<!DOCTYPE`, it's output as a standalone HTML document. Otherwise, it's wrapped in the theme's header and footer.
